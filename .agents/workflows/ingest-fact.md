# 知识吸收: 事实知识 (Ingest Fact)

此工作流指导 Agent 从相关的讨论、设计文档或架构演进中提取纯事实类知识（如架构设计、约束规则、协议说明），并在 Wiki 中创建结构化的知识条目。

1. **上下文提取 (支持 Submodule 感知)**: 
   - 优先通过**当前对话的上下文记录**（即 Agent 在主干项目中了解到的系统设计、约束或事实说明）来提取。
   - 确保所有的读写操作都明确指向 `llm_wiki` 所在的子模块路径。
2. **提炼事实与规则**: 
   - 提取客观事实信息（例如模型的 EP/TP 约束，网络拓扑要求，框架隔离原则）。
   - 说明背后的设计原因及技术背景。
3. **套用模板**: 
   - 使用 `templates/fact_template.md` 将提取的经验进行规范化排版。
   - 务必完整填写 Markdown 顶部的 YAML 元数据。
4. **归档至 Wiki**: 
   - 将格式化后的文档默认保存至 `wiki/05_fact/` 目录下（或根据内容的具体框架/算子属性分类至其它目录），并确保文件名表意清晰（例如 `wiki/05_fact/ep16_constraint_for_2b5.md`）。
   - 如果对应目录不存在，请先创建目录。
5. **更新索引与日志**: 
   - 在 `index.md` 中对应的分类下添加指向新生成的知识条目的双向链接 `[[WikiLink]]`。
   - 在 `changelog.md` / `log.md` 顶部追加一条相关的操作日志，宣告该经验吸收完毕。
